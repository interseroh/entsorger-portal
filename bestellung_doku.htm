Interseroh Entsorger Portal Bestellung API

Beschreibung

Interseroh Entsorger Portal Bestellung API bildet Bestellungen von Interseroh an einen Entsorgungspartner ab. Eine Bestellung ist eine Bedarfsäußerung verbunden mit dem Wunsch auf Leistungserbringung.

Technische Dokumentation

Mehrwert

Ein System, dass den Service Interseroh Entsorger Portal Bestellung API konsumiert kann automatisiert Bestellungen prüfen, beantworten und daraus Aufträge erzeugen.

Funktionalität

Der Service Interseroh Entsorger Portal Bestellung API kann über verschiedene Endpoints genutzt werden. Diese Endpoints sind durch Swagger dokumentiert (Link zu Swagger).
Folgende Funktionalitäten werden bereitgestellt:

  • Abruf einer Liste von Bestellungen
  • Abruf einer einzelnen Bestellung
  • Bestellung beantworten
    • akzeptieren (befindet sich noch in der Umsetzung)
    • ablehnen (befindet sich noch in der Umsetzung)

Der gesamte Service basiert auf dem Datenobjekt Bestellung. Das Objekt enthält Paramter und verweist auf weitere Objekte. Diese verfügen wiederum über eigene Paramter. Im folgenden werden die Objekte und Parameter erläutert:

Objekt Parameter Erläuterung

bestellung		 Eine Bestellung ist eine Bedarfsäußerung verbunden mit dem Wunsch auf Leistungserbringung.

erstellungsdatum Das Datum an dem eine Bestellung erstellt wurde.
faelligkeitsdatum Das Datum der Fälligkeit einer Bestellung.
status Der Status einer Bestellung. Mögliche Werte: OFFEN; DISPONIERT; RUECKGEMELDET; FAKTURIERT; STORNIERT
ticketId Ein eindeutiger, technischer Identifizierer der Bestellung und des gesamten Geschäftsvorganges.
auftrag Ein Kundenauftrag an Interseroh. Durch ihn wird die Bestellung erzeugt.
turnus Gibt an, ob eine Bestellung, durch einen Turnusauftrag (TRUE) erzeugt wurde.
uniqueKey Identifiziert den Auftrag, der eine Bestellung ausgelöst hat.
betriebsstaette Ist eine funktionale Rolle einer [Postalischen Anschrift].Die [Postalische Anschrift] ist dann [Betriebsstätte] wenn sie von einem [Geschäftspartner] betrieben wird.Beschreibung:Darf keine [Postfachadresse] sein.
kennung Eine Fachliche Kennung der Betriebsstätte
lieferant [Geschäftspartner], der uns [Dienstleistungen] oder ein [Handelsgut] liefert. Steht in den meisten Geschäftsvorfällen in kreditorischer Geschäftsbeziehung zur Interseroh.
nummer Fachlicher Identifizierer eines Lieferanten
uniqueKey Technischer Identifizierer eines Lieferanten
detail Beinhaltet verschiedene Details einer Bestellung
dokumentErforderlich Der Vorgang kann erst fakturiert werden nachdem ein optischer Beleg eingereicht wurde. Ein optischer Beleg ist z.B.: ein PDF eines gescannten Wiegescheins.
mitteilung Hier können zusätzliche Informationen hinterlegt werden. Ggf. auch direkt vom Kunden.
retrograd In Ausnahmefällen werden Bestellungen (retrograd=TRUE) nur aus technischen Gründen erzeugt. Diese Bestellungen dürfen nicht gefahren werden. Mögliche Werte: TRUE; FALSE
leistungsadresse Anschrift der Leitungserbringung für den Kunden; unabhängig von der Art der Leistung (Ware, Entsorgung, Dienstleistung)

postalischeAnschrift Ein um einen Adressaten erweiterte [Gebäudeadresse].
adressatenname Name des Adressaten einer postalischen Anschrift
gebaeudeadresse Ein durch eine Adressangabe (oder GPS-Koordinaten) weltweit auffindbares Grundstück. Eine Gebäudeadresse kann mehrere funktionale Rollen annehmen. Die Funktionalität wird aus Sicht des [Geschäftspartner]s bestimmt. Ohne Name, Firma.
ortsbezeichnung Bezeichnung des Ortes einer Gebäudeadresse.
postleitzahl
strassenname
position Postion einer Bestellung
masseinheit Beschreibt die Einheit, mit der die Position gemessen wird.
menge Beschreibt die bestellte Menge einer Position
nummer Jede Position einer Bestellung erhält eine eindeutige Nummer.
typ Mögliche Werte: BAG; CONTAINER; CONTAINER_LEER; FRACHTPAPIERE; FRACTION; FRACTION_2; FREIGHT; HANDLING; SHIP_ONLY_CONTAINER; SHIP_ONLY_FREIGHT; SHIP_ONLY_HANDLING; SHIP_ONLY_FRACTION; VERWERTUNG

uniqueKey Technischer Identifizierer eines Artikels.
avvnummer Die AVV-Nummer des Stoffes, falls der Artikel einen Stoff beschreibt.
