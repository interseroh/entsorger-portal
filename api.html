1. Generelles

In diesem Abschnitt stehen Bedingungen, die alle Anfragen an die API betreffen.


1.1 Authentifizierung

Alle RESTful-Abfragen dieser API setzen voraus, dass ein gültiges OAuth-Token übergeben wird. Wie ein solches Token generiert wird ist hier beschrieben. Dieser Token wird dann, wie hier beschrieben, in den Header des REST-Requests eingefügt.

Einen Test-OAuth-Token kann man unter der URL https://interseroh-portal-test.com/oauth/token erhalten.
In der Produktivumgebung lautet die URL https://h00244l.isd-it.interseroh-ag.local/oauth/token.


2. Bestellungen

In diesem Abschnitt werden API-Aufrufe beschrieben, die den Abruf von Bestellungen zum Gegenstand haben.

Wenn mehrere Bestellungen in der Response zurückgegeben werden, sind die Bestellungen zeitlich sortiert. D.h. die jeweis neueste Bestellung steht an erster Stelle.

Mit dem Test-OAuth-Token kann testweise auf die https://interseroh-portal-test.com/ep-bestellung/v2/bestellungen zugreifen.

Eine Online-Dokumentation zum genauene Aufbau des Request und der Response ist unter der URL http://host:9101/ep-bestellungen/swagger-ui.html verfügbar.


2.1 Alle Bestellungen abrufen

Request

Das Abrufen von Bestellungen wird als GET-Request durchgeführt. der Aufruf gestaltet dich so:

https://www.interseroh-protal.de/ep-bestellung/v2/bestellungen?maxanzahl=xxx

Hierbei ist xxx die Anzahl der Bestellungen, die man maximal abrufen möchte. Wird der Parameter maxanzahl nicht angegeben, werden 100 Bestellungen zurückgegeben.


Response

Bei einem Fehler wird der entsprechende HTTP-Statuscode zurückgesendet. Ist der Statuscode 200, befinden sich die Nutzdaten im Body der Response im JSON-Format.
